# Nexora

## 🧠 Capstone Project: Nexora

### 🔍 Project Overview
Nexora is a full-stack MERN (MongoDB, Express.js, React, Node.js) application designed to enhance student productivity, collaboration, and engagement within the campus ecosystem. By integrating essential academic and community tools into a single platform, Nexora empowers students to stay organized, informed, and connected.

### ✨ Key Features

- **📅 Event Tracking** – Stay updated with campus activities and important dates.
- **📚 Academic Resource Sharing** – Seamlessly upload, access, and share learning materials.
- **📂 File Uploads** – Easily submit assignments or resources with secure storage.
- **🔐 Authentication System** – Standard email/password login with Google OAuth integration.
- **💡 AI-Powered Autocomplete** – Smart task suggestions using LLM-based autocomplete.
- **🔄 Real-Time Updates** – Stay in sync with collaborative changes and updates.
- **✅ Task Manager** – Create, assign, and track academic or personal tasks.
- **👥 Community Features** – Foster peer-to-peer collaboration within the app.

In addition to being a responsive web application, Nexora will also be compiled into an Android APK, ensuring accessibility for students on mobile devices anytime, anywhere.

## 📅 6-Week Development Plan & Timeline

### 🗓️ Week 1: Planning, UI Design & Initial Setup
- Finalize project idea & daily plan. Set up GitHub repository with README and project board.
- Design low-fidelity wireframes (Dashboard, Login, Tasks).
- Create high-fidelity UI designs using Figma.
- Set up GitHub issues & milestones.
- Initialize React frontend with Vite.
- Set up backend architecture using Node.js and Express.
- Push setup work to GitHub and document progress.

### 🗓️ Week 2: Authentication & Core Frontend
- Develop Login/Signup UI in React.
- Implement JWT-based email/password authentication in the backend.
- Integrate frontend and backend authentication logic.
- Add Google OAuth login.
- Implement protected routes in React.
- Deploy backend on Render/Vercel.
- Update Bruno/Postman collection with authentication APIs.

### 🗓️ Week 3: Backend APIs & Database Setup
- Design MongoDB schemas: User, Task, Event, Resource.
- Implement POST API for creating tasks.
- Implement GET API for fetching tasks.
- Implement PUT API for updating tasks.
- Implement DELETE endpoint for task removal.
- Set up schema relationships (e.g., user-task linkage).
- Push API and schema progress to GitHub with documentation.

### 🗓️ Week 4: Frontend Functionality Integration
- Build React components for Task List and Dashboard.
- Connect task-related APIs to frontend.
- Add file upload feature to submit and manage resources.
- Handle backend file uploads using Multer.
- Deploy frontend and conduct initial testing.
- Match frontend design with Figma high-fidelity prototypes.
- Submit frontend implementation and design proof.

### 🗓️ Week 5: AI, Search & Testing
- Integrate LLM-based autocomplete (OpenAI or Replicate API).
- Add task search/auto suggest functionality.
- Configure Jest and write 2 unit tests.
- Expand with 3 additional unit tests.
- Create and test Dockerfile for containerization.
- Prepare for user testing with 5+ student users.
- Push AI, Docker, and testing artifacts to GitHub.

### 🗓️ Week 6: Collaboration, Open Source & Final Wrap-up
- Submit first pull request to open-source repo.
- Submit second open-source PR.
- Submit third open-source PR.
- Invite 3 peers to contribute via PRs to your repository.
- Review and merge incoming pull requests.
- Collect screenshots of app usage from 10–50 users.
- Finalize README, documentation, and submit project deliverables.
